# Ejemplo: LangChain RAG en 15 líneas
from langchain.document_loaders import TextLoader
from langchain.text_splitter import RecursiveCharacterTextSplitter
from langchain.embeddings import OpenAIEmbeddings
from langchain.vectorstores import Chroma

# 1. Cargar documentos
loader = TextLoader("docs/")
docs = loader.load()

# 2. Chunking
splitter = RecursiveCharacterTextSplitter(chunk_size=512, chunk_overlap=50)
chunks = splitter.split_documents(docs)

# 3. Embeddings + Vector Store
db = Chroma.from_documents(chunks, OpenAIEmbeddings())

# 4. Búsqueda
results = db.similarity_search("¿Cómo configuro el firewall?", k=3)
⚠️ LangChain no es necesario: Para proyectos simples, usar directamente la API del LLM + un vector DB es más ligero y mantenible. LangChain brilla en aplicaciones complejas con muchas integraciones.

# Métricas críticas en producción:
1. Latencia (p50, p95, p99) por modelo/endpoint
2. Tokens por request (input + output)
3. Coste por request y por usuario
4. Tasa de error (4xx, 5xx, timeouts)
5. Tasa de "refusal" (modelo rechaza responder)
6. Feedback de usuarios (thumbs up/down)
7. Calidad de respuestas (eval automatizado periódico)
8. Uso de caché (hit rate, ahorro)
